Git and GitHub Guide

What is Git?

  • Git is a version control system that helps track changes in code.
  • It allows multiple people to work on the same project without overwriting each other's work.
  • Git is like a "time machine" for your code â€” you can go back to any previous version.

What is GitHub?

  • GitHub is a platform that hosts your Git repositories online.
  • It helps developers collaborate by sharing and managing their code in one place.
  • Think of GitHub as a social network for developers, where repositories (repos) are like profiles.

Basic Git Commands

1. Initialize a Git Repository:

git init

Sets up Git tracking in your project folder.

2. Check the Status:

git status

Shows changes and the state of the working directory.

3. Add Files for Tracking:

git add <filename>

Stages changes to be committed. Use git add . to add all changes.

4. Commit Changes:

git commit -m "Commit message"

Saves changes with a message describing the updates.

5. Push Changes to GitHub:

git push origin <branch-name>

Sends your local changes to GitHub.

6. Pull Changes from GitHub:

git pull origin <branch-name>

Updates your local repo with changes from GitHub.

7. Create a Branch:

git branch <branch-name>

Makes a new branch for working on features.

Creating a GitHub Pages Site

GitHub Pages allows you to host static websites directly from your GitHub repository. Here's how to set up a simple website:

1. Create a Repository:

  • Go to GitHub and create a new repository.
  • Give it a name, such as my-awesome-site, and make sure it's public.

2. Add Your Website Files:

  • Clone the repository to your local machine using git clone <repo-url>.
  • Add your website files (e.g., index.html, styles.css).
  • Use Git to commit and push the changes:
    • 
            git add .
            git commit -m "Initial commit"
            git push origin main

3. Configure GitHub Pages:

  • Go to your repository on GitHub.
  • Click on Settings, then scroll down to the Pages section.
  • Under "Source," select the branch (e.g., main) and folder (e.g., /root) where your website files are stored.
  • Click Save. Your site will be published at https://your-username.github.io/repo-name.

4. View Your Site:

  • Open the URL provided in the GitHub Pages settings to see your live website.

Advanced GitHub Topics

1. Using Actions for Automation

  • GitHub Actions lets you automate tasks like testing, building, and deploying your projects.
  • Create a workflow file in the .github/workflows directory of your repository.
  • Example workflow to deploy a static site:
    • 
            name: Deploy Static Site
            on:
              push:
                branches:
                  - main
            jobs:
              build:
                runs-on: ubuntu-latest
                steps:
                - name: Checkout Code
                  uses: actions/checkout@v3
                - name: Deploy to GitHub Pages
                  uses: peaceiris/actions-gh-pages@v3
                  with:
                    github_token: ${{ secrets.GITHUB_TOKEN }}
                    publish_dir: ./public

2. Managing Teams and Permissions

  • Create teams within your GitHub organization to manage collaborators efficiently.
  • Assign roles like admin, maintainer, or developer to control access.

3. Protecting Your Branches

  • Enable branch protection rules to prevent direct commits to critical branches like main.
  • Require pull request reviews before merging changes.

4. Using GitHub CLI

  • GitHub CLI (gh) lets you perform GitHub operations directly from your terminal.
  • Install the CLI and log in using gh auth login.
  • Example: Create a new repository:
    • gh repo create my-new-repo --public

5. GitHub Codespaces

  • Codespaces provide a cloud-based development environment directly in GitHub.
  • Start coding in seconds without setting up your local machine.

Additional Tips

  • Keep your commits small and descriptive.
  • Use README.md to document your project.
  • Regularly update and maintain your repositories to keep them relevant.
  • Explore GitHub's marketplace for integrations and tools to enhance your workflow.
